# Crawl Query Policy

## 핵심 원칙

네이버 뉴스 검색 API에는 공식 날짜 범위 파라미터가 없다. 수집 기간은 query 문자열이 아니라 API 응답의 `pubDate`를 파싱한 뒤 `date_from <= pubDate <= date_to` 조건으로 후처리한다.

자동 수집, 2026년 1월 테스트 수집, 2024~2026년 5월 Backfill, 주간 증분 업데이트는 모두 월 키워드를 query에 붙이지 않는다. `2026년 1월`, `2026.01`, `2026-01` 같은 날짜 문자열은 기사 제목/본문에 없을 수 있어 검색 recall을 낮출 수 있다.

## 자동 검색어 계층

자동 QueryGenerator는 다음 계층만 사용한다.

1. 공통 신상품 검색어: `보험 신상품`, `보험상품 출시`, `보험사 신상품`, `장기보험 출시`, `보장성보험 출시` 등
2. 상품군별 검색어: `건강보험 출시`, `암보험 출시`, `실손보험 출시`, `자동차보험 출시`, `펫보험 출시`, `연금보험 출시` 등
3. 회사별 검색어: `{company} 신상품`, `{company} 보험 출시`, `{company} 보장성보험`, `{company} 출시했다`, `{company} 선보였다` 등
4. 회사 × 상품군 검색어: `{company} 건강보험`, `{company} 운전자보험`, `{company} 펫보험`, `{company} 종신보험` 등
5. 상품명 후속 검색어: DB에서 이미 발견된 표준 상품명과 `회사명 + 상품명`

특정 회사와 특정 상품명 조합을 코드에 하드코딩하지 않는다. 이미 DB에서 발견된 상품명은 follow-up query로 사용할 수 있다.

## 기간 필터

수집기는 `sort=date`, `display=100`, `start=1,101,201,...,901` 방식으로 페이지를 조회한다. 각 item의 `pubDate`를 파싱해 기간 안에 있으면 저장하고, 범위 밖이면 `articles_out_of_range`를 증가시킨다.

중복 제거는 `original_url`, `link`, `title + pubDate + publisher` 순서의 hash 기준으로 처리한다.

## 관리자 Preview

관리자 진단용 `POST /api/admin/search-preview/naver-news`는 임의 검색어를 허용한다.

예:

```json
{
  "query": "한화손해보험 건강보험 2026년 1월",
  "display": 10,
  "sort": "date"
}
```

이 preview는 검색 품질을 눈으로 확인하기 위한 기능이다. 자동 배치 검색어에는 월 키워드를 추가하지 않는다.

## 누락 보정

검색 API에서 누락된 URL은 관리자 URL 직접 등록이나 별도 수동 수집 흐름으로 보정한다. 자동 월 키워드 확장은 누락 보정 수단으로 사용하지 않는다.
